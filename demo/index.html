iron-overlay can be used to implement a variety of UI widgets that appear on top of other elements.

It may be cancelled or closed. Both actions dismiss the dialog. Closed implies that the user took explicit action to dismiss the dialog, for example by clicking a button in the dialog, and cancelled is an implicit dismissal. By default, a iron-overlay is cancelled when the user clicks outside the element or by when the user hits the ESC key.

Dialogs

Dialogs appear on top of other elements. By default, iron-overlay is centered in the viewport.

Hello world!

iron-overlay constrains its size to fit inside the viewport. To implement a dialog that scrolls on overflow, either make it or a child element `overflow:auto`. You can also set the `margin` property to set margins around the dialog.
